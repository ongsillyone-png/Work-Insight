# แผนการสร้างข้อมูลจำลอง (Mock Activities Generation Plan)

นี่คือบันทึกข้อตกลงและเงื่อนไขทั้งหมดที่เราได้เขียนไว้ในสคริปต์ `scripts/generate_mock_activities.js` เพื่อเตรียมไว้ใช้สร้างข้อมูลให้กับพนักงานคนต่อๆ ไปในวันพรุ่งนี้ครับ

## 📝 เงื่อนไขและกฎการสุ่มข้อมูล (Rules & Logic)

1. **ช่วงวันที่**: วันที่ 1 มกราคม 2569 จนถึงปัจจุบัน
2. **วันหยุด**: จะไม่สุ่มงานในวันเสาร์-อาทิตย์ และวันที่ระบุในตาราง `holiday`
3. **อัตราส่วนงาน (60/40 Rule)**: 
   - รับคีย์เวิร์ดหน้างานหลัก (Core Keywords) ของแต่ละคน
   - 60% ของงานที่ถูกสุ่มจะเป็นงานหลัก (Core Activities)
   - 40% จะเป็นการสุ่มจากงานอื่นๆ ที่มีสิทธิ์ทำ (Other Activities)
4. **ระยะเวลาทำงานต่อวัน**: สุ่มเวลาทำงานต่อวันให้รวมกันได้ **5 ถึง 7 ชั่วโมง (300-420 นาที)**
5. **จำนวนกิจกรรมต่อวัน**: บังคับให้อยู่ที่ **3 ถึง 6 กิจกรรม** โดยเฉลี่ยเวลาให้ลงตัว
6. **การกระจายช่วงเวลา (Session)**: 
   - กิจกรรมที่เกิดขึ้นในช่วง 3 ชั่วโมงแรกของวัน จะถูกระบุเป็น "เช้า (Morning)"
   - กิจกรรมที่เหลือของวัน จะถูกระบุเป็น "บ่าย (Afternoon)"
7. **เวลาบันทึก (created_at / updated_at)**:
   - `created_at` สุ่มเวลาภายในกรอบ Session (เช่น เช้า สุ่มระหว่าง 8-11 โมง)
   - `updated_at` จะบวกเวลาเพิ่มจาก `created_at` ไปอีก 0-5 นาที ให้ดูเป็นธรรมชาติ
   - นำคำว่า "Mock Data" ในช่องหมายเหตุออกแล้ว เพื่อความแนบเนียน
8. **กฎพิเศษ (Exceptions)**:
   - **งาน "นอกสถานที่"**: บังคับให้เป็นช่วง "ทั้งวัน (AllDay)", ให้เวลาเต็ม 7 ชม. (420 นาที), ไม่มี Location และถือเป็นงานเดียวของวันนั้น
   - **งาน "ประชุม"**: ถูกจำกัดให้สุ่มได้ **ไม่เกินเดือนละ 3 วัน** และการประชุมแต่ละครั้งต้อง **ห่างกันอย่างน้อย 7 วัน**

## 📍 กฎการระบุสถานที่ทำงาน (Smart Location Assignment)

- **ห้องประชุม (ID: 6)**: สำหรับงานที่มีคำว่า `ประชุม`, `ถ่ายภาพ`, `โสตทัศนะ`, `นำเสนอ`
- **ห้องเซิร์ฟเวอร์ (ID: 5)**: สำหรับงานที่มีคำว่า `network`, `server`, `เซิร์ฟเวอร์`, `เน็ตเวิร์ค`
- **ห้องกลุ่มงานสุขภาพดิจิทัล (ID: 7)**: สำหรับงานที่มีคำว่า `เอกสาร`, `แผน`, `โปรแกรม`, `เขียน sql`
- **สุ่มจุดบริการต่างๆ (ยกเว้น ID 5, 6, 7)**: สำหรับงานที่มีคำว่า `ซ่อม`, `ปริ้นเตอร์`, `คอม`, `เครือข่าย`
- นอกเหนือจากนี้: สุ่ม Location ทั่วไป

## 🚀 ขั้นตอนการทำงานสำหรับวันพรุ่งนี้

เมื่อคุณพร้อมทำข้อมูลของพนักงานคนถัดไป ให้เตรียมข้อมูล 2 ส่วนนี้มาบอกผมได้เลยครับ:
1. **User ID** (เช่น ID 3, 4, 5...)
2. **คำค้นหาหลัก / หน้างานประจำ** (เช่น งานบัญชี, ประสานงาน, ตรวจสอบเอกสาร) 

จากนั้นผมจะตั้งค่าและสั่งรันสคริปต์ให้ทันทีครับ ฝันดีครับ พรุ่งนี้ค่อยลุยกันต่อ!
